# 热点雷达：全网热搜抓取 + 行业筛选 + 创作引导

## 执行回执（硬规则）

只要本轮因为 `skill` / `技能` 指名，或因为关键词命中而进入本技能，第一行先写：
`已走技能：热点雷达`

若当前还只是信息收集阶段，则写：
`已走技能：热点雷达（信息收集阶段）`

---

## Step 0: 确认行业和城市

**需要的信息**：行业、城市、热点范围（全国/本地/全都要）。

从系统记忆、用户画像、对话上下文中提取以上信息。已有的直接用，缺的才问。如果都缺，**一次问完，不拆成多轮**：

### 提问（精简版，一次问完）

```
开始之前确认3个信息，方便帮你精准筛选热点：

1. 你是什么行业？（如：餐饮、美业、教育、汽车、制造业...）
2. 你在哪个城市？（如：北京、武汉、深圳，或填"不限"）
3. 你想看哪个范围的热点？
   1. 全国热点（默认）
   2. 本地热点
   3. 全都要

直接回复就行，比如"餐饮，武汉，3"
```

收到回复后，记住这三个信息，后续本轮对话内不再重复问。

## Step 1: 抓取全网热搜

### 1.1 全国热搜（必做）

用 WebFetch 访问 `https://tophub.today/`，提取以下平台的热搜 Top 10：
- 微博热搜
- 抖音热点
- 知乎热榜
- 百度热搜

**WebFetch prompt**：
```
提取页面上所有平台的热搜排行榜数据，包括：
1. 微博热搜 Top10（排名、话题名称、热度数字）
2. 抖音热点/抖音热榜 Top10
3. 知乎热榜 Top10
4. 百度热搜 Top10
5. 其他平台如有也提取

每条数据包含：排名、标题、热度指数（如有）、所属平台
```

### 1.2 本地热点（仅当用户选了"本地"或"全都要"时执行）

用 WebSearch 搜索：`{用户城市} 今日热点新闻`

提取本地相关的新闻和事件，整理为 Top 5。

### 1.3 全球热点（仅当用户明确要求时执行）

用 WebSearch 搜索：`today global trending news` 或 `国际热点新闻 今日`

## Step 2: AI 筛选 + 输出简报

基于 Step 0 获取的用户行业信息，对所有热搜数据做智能筛选。

### 筛选逻辑

1. **跨平台去重**：同一事件在多平台出现的，合并为一条，标注出现平台数
2. **大热点判定**：出现在2个以上平台，或单平台热度排名前3
3. **行业关联判定**：
   - 强关联：热点话题直接涉及用户行业关键词
   - 可借势：话题可以通过观点输出、类比、反差等方式接入用户行业
   - 无关：跳过不展示

### 编号规则（硬规则）

所有热点从 1 开始连续编号，跨分区不重置。例如：
- 全网大热点：1、2、3、4、5
- 行业相关：6、7、8、9、10
- 本地热点：11、12、13、14、15

用户只需回复一个数字即可选择。

### 输出格式（严格遵守）

```
📰 今日热点简报（{行业} · {城市}）
━━━━━━━━━━━━━━━━━━

🔥 全网大热点

1. {热点标题}
   📊 热度：{数字} | 📍 来源：{平台1、平台2}

2. {热点标题}
   📊 热度：{数字} | 📍 来源：{平台}

3. ...
4. ...
5. ...

🎯 跟你行业相关

6. {热点标题}
   💡 关联理由：{一句话}
   🔗 借势方向：{一句话}

7. ...
8. ...
9. ...
10. ...

{如果用户选了本地热点}
📍 {城市}本地热点

11. {本地新闻标题}
    💡 本地关联：{一句话说明本地生意怎么用}

12. ...
13. ...
14. ...
15. ...

━━━━━━━━━━━━━━━━━━
👉 回复编号选择热点，如"6"或"选6、11"
```

### 输出规则

- 全网大热点固定出 5 条
- 行业相关出 3-5 条（不足则补弱关联，标注"弱关联"）
- 本地热点出 3-5 条（仅当用户选了本地时）
- 每条热点标题不超过25字，超出截断
- 关联理由和借势方向各一句话，不超过30字
- 如果完全找不到行业相关热点，告诉用户"今天没有强相关热点，以下是可借势的方向"

## Step 3: 用户选择热点 + 深挖详情

### 3.1 接收选择 + 搜索详情

用户回复编号后（如"6"、"选11"、"6和11"等），**先用 WebSearch 搜索每个选中热点的详细信息**：

搜索内容包括：
- 事件的完整经过和关键事实
- 核心数据和关键数字
- 各方反应和争议点
- 最新进展

**支持一次选多个**：用户说"6和11"或"选6、11"，则同时深挖多个热点，每个热点各发一次 WebSearch。

### 3.2 展示事实摘要

将搜索到的详细信息整理为**简要事实摘要**（每个热点 5-8 句话），展示给用户。

格式：
```
## 你选的热点 · 事实摘要

### 热点{N}：{标题}

**事实**：{5-8句话的事实摘要，包含关键数据和争议点}

---

### 热点{M}：{标题}

**事实**：{5-8句话的事实摘要}
```

### 3.3 询问创作方向

事实摘要展示后，询问用户想怎么用：

```
想用这些热点做什么？

1. 写朋友圈文案（1-3条）
2. 写短视频口播脚本
3. 写投流素材脚本
4. 写公众号文章
5. 其他（你说）

可以每个热点选不同方向，也可以统一选一个。
```

## Step 4: 执行创作

根据用户选择的创作方向执行。

### 选 1：朋友圈文案

- 读取用户知识库中的朋友圈SOP（如有）
- 每个热点产出 2-3 条朋友圈文案
- 每条包含：文案正文 + 建议配图方向
- 风格：有观点、有态度、大白话、真诚不做作
- 结构：热点事实引入 → 个人观点/行业洞察 → 自然收尾（不硬广）

### 选 2：短视频口播脚本

检查环境中是否有「热点文案改写」Skill：
- 有 → 提示用户：`你可以说"热点文案改写"来生成完整的短视频脚本`，并把热点事实摘要传过去
- 没有 → 在当前 Skill 内完成，结构：
  1. 3秒判词开场（直接态度，制造反差/悬念）
  2. 30秒画面还原（人物+动作+关键话）
  3. 底层机制解释（用比喻让人秒懂）
  4. 三条建议（前两条给认知，第三条接业务）
  5. 金句收口 + 低压引导

### 选 3：投流素材脚本

- 产出 3 版不同切入角度的投流短脚本（每版 300-500 字）
- 强调痛点前置、3秒留人、强钩子开头
- 每版标注适合的投放场景（信息流/搜索/DOU+）

### 选 4：公众号文章

检查环境中是否有「公众号文章」Skill：
- 有 → 提示用户调用
- 没有 → 产出：文章标题3选1 + 完整大纲 + 开头300字

### 选 5：其他

根据用户描述灵活执行。

## 通用规则

### 零配置，即插即用
- 不需要 API Key
- 不需要环境变量
- 不需要登录任何平台
- 不需要安装任何依赖
- 只依赖 WebFetch 和 WebSearch，所有主流 AI 工具都支持
- 复制整个文件夹到任何电脑，直接可用

### 行业适配
- 本 Skill 适用于任何行业的用户
- 行业筛选完全由 AI 根据用户回答动态完成
- 不硬编码任何行业关键词
- 不同行业的用户看到的行业相关热点完全不同

### 创作风格（通用默认）
- 大白话，不用书面语
- 有观点、有态度
- 80%价值输出 + 20%业务植入（如果用户有明确业务）
- 不散播焦虑，不制造恐慌
- 如果用户知识库有写作风格规范，以用户的规范为准

### 禁止操作

**严禁** 编造热度数据或热搜排名——抓不到就写"热度数据暂无"，不能自己编数字。
**严禁** 把与用户行业完全无关的热点强行标为"行业相关"——宁可少给，不硬蹭。
**严禁** 在创作输出中散播焦虑、制造恐慌或使用极端情绪化表达。
**严禁** 在事实摘要中添加未经搜索验证的信息——搜到什么写什么，不加戏。

### 异常处理
- WebFetch 抓取 tophub.today 失败 → 改用 WebSearch 搜索"今日热搜榜 微博 抖音 知乎 百度"作为备选
- 某平台数据缺失 → 跳过该平台，用已有数据继续
- 本地热点搜不到 → 告诉用户"今天暂无突出的本地热点"，继续展示全国热点
- 用户没有业务画像也不愿回答 → 只输出全网大热点 Top10，跳过行业筛选

## 简报保存

创作完成后（Step 4 执行完毕），询问用户是否保存：

```
要把今天的热点简报存一份吗？后续回看或二次创作方便。

1. 保存（推荐）
2. 不用了
```

### 保存规则（主动追问）

热点简报展示完后，**主动问一次**：

```
---

📁 **要把这份热点简报保存到你的本地内容库吗？**

  会保存到：我的内容库（勿动）/04-热点文案/{YYYY-MM-DD}-热点简报.md

  1. 保存（推荐）
  2. 不用
```

**规则**：
- 用户选"1" → 立刻调用文件写入工具真实保存，确认后回复"✅ 已保存到 {完整路径}"
- 用户选"2" → 结束
- 用户模糊回答（"好""嗯""可以"）→ 按"1"处理
- **严禁** 跳过这一问
- **严禁** 只在对话里说"已保存"但实际没写文件

**保存细节**：
- 文件名：`{YYYY-MM-DD}-热点简报.md`（当天日期）
- 保存位置：`~/Desktop/我的内容库（勿动）/04-热点文案/`（和热点文案归到同一个文件夹）
- 文件夹不存在：告诉用户"你还没搭建内容库，先跑一下'内容库搭建'skill"
- 版本管理：同一天重新生成，文件名加时分：`{YYYY-MM-DD-HH:MM}-热点简报.md`
- 保存内容：完整热点简报 + 用户选择的热点事实摘要 + 创作产出
